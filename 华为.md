####	配置链路振荡保护功能

如果在链路振荡时间间隔内，链路振荡次数达到了设定值，则将该接口关闭，记录为**ERROR DOWN(link-flap)**状态。缺省情况下，使能链路振荡保护功能后，10秒内接口Up/Down状态变化5次，接口就将变为Down状态。

```bash
int g 0/0/1
	port link-flap protection enable
	port link-flap interval 10	# 配置接口的链路振荡时间间隔，缺省为10秒
	port link-flap threshold 5	# 配置接口的链路振荡次数，缺省为5次
```

当处于Error-down状态的接口数量较少，且需要强制开启接口时，可在该接口视图下依次执行命令[**shutdown**](http://127.0.0.1:51299/icslite/hdx/pages/HDXAZI02127_08_zh/HDXAZI02127_08_zh/resources/dc/cmdqueryname=shutdown)和[**undo shutdown**](http://127.0.0.1:51299/icslite/hdx/pages/HDXAZI02127_08_zh/HDXAZI02127_08_zh/resources/dc/cmdqueryname=undo+shutdown)；也可以执行命令[**restart**](http://127.0.0.1:51299/icslite/hdx/pages/HDXAZI02127_08_zh/HDXAZI02127_08_zh/resources/dc/cmdqueryname=restart)，重启接口。

####	配置光功率低触发 Error-down

接口光功率低时，会出现业务丢包等故障。可配置接口光功率低触发Error-down功能，当接口光功率低于设置的告警下限阈值时，系统将改接口关闭，记录为ERROR DOWN（transceiver-power-low）。

###	VLAN聚合（Sub-VLAN）

![fig_dc_cfg_super-vlan_001601xs](E:\Photo_Wall\fig_dc_cfg_super-vlan_001601xs.png)

配置思路：

1. 在SWB上启用Super-VLAN的Proxy ARP功能，使得不同用户间三层互通。

```bash
# 配置Super-VLAN 4 将VLAN2、3加入到Super-VLAN4中
[SWB] vlan 4
[SWB-vlan4] aggregate-vlan
[SWB-vlan4] access-vlan 2 to 3
```

如果只让某些Sub-VLAN之间互通，可以通过在主机上配置静态ARP来实现，将访问目的主机的IP地址与网关MAC地址绑定。



###	VLAN终结

####	定义

VLAN是指设备对接收到的报文中的VLAN标签进行识别，根据后续的转发行为对报文的单层或双层VLAN标签进行剥离，然后进行三层转发或其他处理。

####	分类

根据对所终结的VLAN报文处理方式的不同，VLAN终结分为以下两种：

- Dot1q终结：对接收到的带有一层或两层VLAN Tag的报文，剥除报文的最外一层VLAN Tag；对从接口发出的报文，添加一层VLAN Tag。
- QinQ终结：既剥除两层；增加两层。



####	目的

划分VLAN后，VLAN内的主机可以二层互通，而VLAN间的主机不能二层互通，可以在三层交换机上通过VLANIF来实现VLAN间的三层互通。但如图所示，当三层交换机的三层接口有限时，而三层接口又无法关联VLANIF。此时可将一个三层以太网接口虚拟成多个逻辑子接口。

![fig_dc_cfg_vlan-termination_000201](E:\Photo_Wall\fig_dc_cfg_vlan-termination_000201.png)

而三层以太网子接口不支持VLAN报文，当它收到VLAN报文会丢弃，所以需要在子接口上将VLAN Tag剥离掉，也就是需要VLAN终结。

###	GVRP 

####	定义

GARP 协议主要是用于建立一种属性传递扩散机制。GVRP 是 GARP 的一种应用，用于注册和注销VLAN属性。

####	目的

如果需要为网络中的所有设备都配置VLAN，就需要管理员在每台设备上分别进行手工添加。如果遇到组网复杂，管理员无法短时间了解网络拓扑，或者是网络VLAN太多，工作量会非常大且容易出错。在这种情况下，用户可以通过GVRP的VLAN自动注册功能完成VLAN的配置。

####	基本概念

注册模式：

手工配置的VLAN称为静态VLAN，通过GVRP协议创建的VLAN称为动态VLAN。GVRP有三种注册模式：

- Normal模式：允许动态VLAN在端口上进行注册，同时会发送静态VLAN和动态VLAN的声明信息。
- Fixed模式：不允许动态VLAN在端口上注册，只发送静态VLAN的声明信息。
- Forbidden模式：不允许动态VLAN在端口上进行注册，同时删除端口上除了VLAN1外的所有VLAN，只发送VLAN1的声明信息。

###	VCMP

####	定义

VLAN集中管理协议VCMP（VLAN Central Management Protocol）是位于二层的通信协议。它提供了二层网络中传播VLAN配置信息的功能。

####	目的

大型企业网中，交换机很多，会有大量的VLAN信息需要配置和维护，如果仅靠网络管理员手工操作，工作量很大，也不能保障配置的一致性。

为了解决上述问题，可通过VCMP实现VLAN的集中管理，这样，只需在一台交换机上进行创建，删除VLAN等操作，这些变更会自动通知到指定范围的所有交换机，从而使这些交换机无需手工操作即可实现VLAN的创建、删除等动作的同步，即减少了在多台交换机上修改同一个数据的工作量，也保证了修改的一致性。

####	基本概念

VCMP使用域来管理交换机，用角色来确定设备的属性。

| 角色        | 定义及作用                    | 说明                                                         |
| ----------- | ----------------------------- | ------------------------------------------------------------ |
| Server      | VCMP 管理域的管理角色         | Server 上创建、删除和修改VLAN会在全域传播                    |
| Client      | VCMP 管理与的被管角色         | Client 上创建、删除和修改VLAN不会在域内传播，会被Server发送的信息覆盖 |
| Transparent | 透传角色，透明状态            | Transparent仅仅只会转发VCMP报文                              |
| Silent      | VCMP 管理域的边缘，用来隔离域 | Silent 收到 VCMP报文后直接丢弃                               |



####	配置案例

![fig_dc_cfg_vcmp_001702](E:\Photo_Wall\fig_dc_cfg_vcmp_001702.png)

由于 VCMP报文只能在 Trunk 和 Hybrid 类型接口上传送，为免去手工配置链路类型的麻烦，部署VCMP时，一般需要同时部署LNP动态协商链路类型，最大程序简化用户配置。

1. 配置通过LNP自动协商链路类型

   缺省情况下，全局和接口上的LNP处于使能状态。可以使用 `display lnp summary` 

2. 指定各设备的角色

   ```bash
   [AGG] vcmp role server
   [ACC1] vcmp role silent
   [ACC2] vcmp role client
   [ACC3] vcmp role transparent
   ```

3. Server和Client上配置VCMP相关参数

   ```bash
   # 在AGG上配置VCMP管理域、设备ID和认证密码。
   [AGG] vcmp domain vdl
   [AGG] vcmp device-id server
   [AGG] vcmp authentication sha2-256 password Hello
   # 在ACC2上配置VCMP管理域和认证密码。
   [ACC2] vcmp domain vd1
   [ACC2] vcmp authentication sha2-256 password Hello
   # 在ACC3上配置VCMP管理域和认证密码。
   [ACC3] vcmp domain vd1
   [ACC3] vcmp authentication sha2-256 password Hello
   ```

4. 使能VCMP功能

   ```bash
   # 缺省情况下，接口上的VCMP功能已使能，无需再使能。但为避免VCMP报文影响PC终端，可Client连接PC终端接口上去使能 
   # VCMP 功能
   [ACC2] interface GigabitEthernet 0/0/2

   [ACC2-GigabitEthernet0/0/2] quit
   [ACC3] interface GigabitEthernet 0/0/2
   [ACC3-GigabitEthernet0/0/2] vcmp disable
   [ACC3-GigabitEthernet0/0/2] quit
   ```

5. 验证配置

   ```bash

   VCMP information:
   Domain                  : vd1
   Role                    : Server
   Server ID               : server
   Configuration Revision  : 0x239c0000
   Password                : ******
   ```

###	MSTP配置

####	基本概念

STP/RSTP存在同一个缺陷：由于局域网内所有VLAN共享一颗生成树，因此无法在VLAN间实现数据流量的负载均衡，链路被阻塞后将不承载任何流量，还有可能造成部分VLAN的报文无法转发。

为了弥补缺陷，MSTP把一个交换网络划分成多个域，每个域内形成多颗生成树，生成树之间彼此独立，每颗生成树叫做一个多生成树实例MSTI（Multiple Spanning Tree Instance），每个域叫做一个MST域（MST Region：Multiple Spanning Tree Region）

![fig_dc_fd_stp_001102](E:\Photo_Wall\fig_dc_fd_stp_001102.png)

MSTP 向下兼容 STP/RSTP

####	专有概念

- **MST域（MST Region）**

  MST域是多生成树域，由多台交换机以及他们之间的网段构成。同一MST域的设备具有下列特点：

  - 具有相同的域名
  - 具有相同的VLAN到生成树实例映射配置
  - 具有相同的MSTP修订级别

- **VLAN映射表**

  描述VLAN和MSTI之间的映射关系

- **CST公共生成树**

  Common Spanning Tree 是连接交换机网络内所有MST域的一颗生成树

- **端口角色**

  - 根端口
  - 指定端口
  - Alternate端口
  - Backup端口
  - Master端口
  - 域边缘端口
  - 边缘端口

####	配置举例

配置MSTP基本功能

![image-20260514134607044](C:\Users\46823\AppData\Roaming\Typora\typora-user-images\image-20260514134607044.png)

配置思路：

1. 在处于环形网络中的设备上配置MSTP基本功能，与PC相连的端口不用参与MSTP计算，将其设置为边缘端口。
2. 配置保护功能，实现对设备或链路的保护，比如在根桥的指定端口配置根保护功能。



1.配置 MST域，使得设备都在一个域中，并配置一样的vlan映射表

```bash
# 缺省域配置是VLAN 0~4094 都映射到 instance 0里
[SwitchA] stp region-configuration
[SwitchA-mst-region] region-name RG1
[SwitchA-mst-region] instance 1 vlan 2 to 10
[SwitchA-mst-region] instance 2 vlan 11 to 20
[SwitchA-mst-region] active region-configuration
```

2.配置 MSTI1 与 MSTI2 的根桥和备份根桥

```bash
[SWA] stp instance 1 root primary
[SWA] stp instance 2 root secondary
```

3.配置实例中MSTI1和MSTI2中将要被阻塞的路径开销值大于缺省值

```
# 配置端口路径开销值的计算方法为华为计算法
[SWC] stp pathcost-standard legacy
[SWC] interface g 0/0/2
[SWC-G0/0/2] stp instance 2 cost 20000
```

4.将于终端相连的端口设置为边缘端口

```bash
[SWC] interface g 0/0/1
[SWC-G0/0/1] stp edged-port enable
# 配置BPDU保护功能
[SWC] stp bpdu-protection
```

5.配置保护功能，如在各实例的根桥设备的指定端口配置根保护功能

```bash
[SWA] int g 0/0/1
[SWA-G0/0/1] stp root-protection
```

###	Loopback Detection配置

####	基本概念

网络中的环路会导致设备对广播、组播以及未知单播等报文进行重复发送，造成网络资源浪费甚至网络瘫痪。为了能够及时发现二层网络中的环路，避免对整个网络造成严重影响，需要提供一种检测技术，使网络中出现环路时能及时通知用户检查网络连接和配置情况，并能够将出问题的接口置于某种受控状态。

###	ARP配置

####	配置静态ARP

对于以下场景，用户可以配置静态ARP表项：

- 对于网络中的重要设备，如服务器等，可以在交换机上配置静态ARP表项，这样可以避免交换机上重要设备IP地址对应的ARP表项被ARP攻击报文错误更新，从而保证用户与重要设备之间正常通信。
- 当网络中用户设备的MAC地址为组播MAC地址时，可以在交换机上配置静态ARP表项。缺省情况下，设备收到源MAC地址为组播MAC地址的ARP报文时不会进行ARP学习。
- 当希望禁止某个IP地址访问设备时，可以在交换机上配置静态ARP表项，将该IP地址与一个不存在的MAC地址进行绑定。

####	配置举例

![fig_dc_s_ccase_arp_00101ds](E:\Photo_Wall\fig_dc_s_ccase_arp_00101ds.png)



在Switch上配置静态ARP表项

```bash
[SW] arp static 10.164.1.1 00e0-fc01-0001 vid 10 interface g 1/0/1		// 为总裁办公室主机配置静态arp
[SW] arp static 10.164.10.1 00e0-fc02-1234 interface g 1/0/2			// 为文件服务器配置静态arp
```



####	配置路由式Proxy ARP示例

企业内部进行子网划分时，可能会出现两个子网网络属于同一网段，当时却不属于同一物理网络的情况，两个子网网络间被交换机分隔。这时可以通过修改网络内主机的路由信息，使发往其他子网的数据先发送到连接不同子网的网关设备上，再由网关设备转发此数据报文。但是这种解决方案需要配置子网中所有主机的路由，并不便于管理和维护。

在网关上部署路由式Proxy ARP功能，可以有效解决子网划分带来的管理和维护方面的问题。路由式Proxy ARP可以使IP地址属于同一网段却不属于同一物理网络的主机间能够相互通信，并且主机上不需要配置缺省网关，便于管理和维护。



####	配置举例

![fig_dc_s_ccase_arp_00201ds](E:\Photo_Wall\fig_dc_s_ccase_arp_00201ds.png)



某企业的子公司A和子公司B位于不同城市，且所使用的IP地址部署为同一个网段（172.16.0.0/16）。连接子公司A的Switch_1与连接子公司B的Switch_2之间路由可达。由于两个子公司之间被设备间隔，属于不同的广播域，因此无法在同一个局域网内实现互通；子公司的主机没有配置默认网关，无法实现跨网段互通。现在公司需要在不改变主机配置的情况下，实现两个子公司之间的通信。

配置路由式 Proxy ARP：

```bash
[SW1-Vlanif10] arp-proxy enable
[SW2-Vlanif20] arp-proxy enable
```

###	DHCP配置（基于接口地址池）

DHCP服务器通过地址池为用户分配IP地址等网络参数。地址池分为接口地址池和全局地址池。

![fig_dc_s_case_dhcp_00101](E:\Photo_Wall\fig_dc_s_case_dhcp_00101.png)



**组网需求：**

某企业为办公终端规划了两个网段，网段10.1.1.0/24内PC为员工固定办公终端，网段10.1.2.0/24供企业出差人员临时接入网络。企业希望通过DHCP方式为员工固定办公终端和企业出差人员分配IP地址。其中，企业管理者的办公PC（Client_1）由于业务需要，希望使用固定IP地址为10.1.1.100/24。

**操作步骤：**

1.使能DHCP服务，缺省未使能

```bash
[SW] dhcp enable
```

2.配置接口地址池

```bash
[SW] interface vlanif 10
[SW-Vlanif10] dhcp select interface
[SW-Vlanif10] dhcp server lease day 30
[SW-Vlanif10] dhcp server static-bind ip-address 10.1.1.100 mac-address 00e0-fc12-3456
```

3.配置DHCP数据保持功能，设置发生故障时，可在系统重启后，执行dhcp server database recover，从存储设备文件恢复dhcp 数据。

```bash
[SW] dhcp server database enable 
```

4.验证配置结果

```bash
[SW] display ip pool interface vlanif 10
```



###	DHCP配置（基于全局地址池）





**组网需求：**

某企业有两个办公室，为了节省资源，两个办公室内的主机由Switch作为DHCP服务器统一分配IP地址。办公室1所属的网段为10.1.1.0/25，主机都加入VLAN10，地址租期为10天；办公室2所属的网段为10.1.1.128/25，主机都加入VLAN11，地址租期为2天。

```bash
[SW-ip-pool-pool1] gateway-list 10.1.1.1
[SW-ip-pool-pool1] lease day 10
```

2.使能DHCP服务器

```bash
[SW] int vlanif 10
[SW-Vlanif10] dhcp select global
```

3.验证配置结果

```bash
[SW] display ip pool name pool1
```
