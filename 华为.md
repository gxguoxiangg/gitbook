####	配置链路振荡保护功能

如果在链路振荡时间间隔内，链路振荡次数达到了设定值，则将该接口关闭，记录为**ERROR DOWN(link-flap)**状态。缺省情况下，使能链路振荡保护功能后，10秒内接口Up/Down状态变化5次，接口就将变为Down状态。

```bash
int g 0/0/1
	port link-flap protection enable
	port link-flap interval 10	# 配置接口的链路振荡时间间隔，缺省为10秒
	port link-flap threshold 5	# 配置接口的链路振荡次数，缺省为5次
```

当处于Error-down状态的接口数量较少，且需要强制开启接口时，可在该接口视图下依次执行命令[**shutdown**](http://127.0.0.1:51299/icslite/hdx/pages/HDXAZI02127_08_zh/HDXAZI02127_08_zh/resources/dc/cmdqueryname=shutdown)和[**undo shutdown**](http://127.0.0.1:51299/icslite/hdx/pages/HDXAZI02127_08_zh/HDXAZI02127_08_zh/resources/dc/cmdqueryname=undo+shutdown)；也可以执行命令[**restart**](http://127.0.0.1:51299/icslite/hdx/pages/HDXAZI02127_08_zh/HDXAZI02127_08_zh/resources/dc/cmdqueryname=restart)，重启接口。

####	配置光功率低触发 Error-down

接口光功率低时，会出现业务丢包等故障。可配置接口光功率低触发Error-down功能，当接口光功率低于设置的告警下限阈值时，系统将改接口关闭，记录为ERROR DOWN（transceiver-power-low）。

###	VLAN聚合（Sub-VLAN）

![fig_dc_cfg_super-vlan_001601xs](E:\Photo_Wall\fig_dc_cfg_super-vlan_001601xs.png)

配置思路：

1. 在SWB上启用Super-VLAN的Proxy ARP功能，使得不同用户间三层互通。

```bash
# 配置Super-VLAN 4 将VLAN2、3加入到Super-VLAN4中
[SWB] vlan 4
[SWB-vlan4] aggregate-vlan
[SWB-vlan4] access-vlan 2 to 3
```

如果只让某些Sub-VLAN之间互通，可以通过在主机上配置静态ARP来实现，将访问目的主机的IP地址与网关MAC地址绑定。



###	VLAN终结

####	定义

VLAN是指设备对接收到的报文中的VLAN标签进行识别，根据后续的转发行为对报文的单层或双层VLAN标签进行剥离，然后进行三层转发或其他处理。

####	分类

根据对所终结的VLAN报文处理方式的不同，VLAN终结分为以下两种：

- Dot1q终结：对接收到的带有一层或两层VLAN Tag的报文，剥除报文的最外一层VLAN Tag；对从接口发出的报文，添加一层VLAN Tag。
- QinQ终结：既剥除两层；增加两层。



####	目的

划分VLAN后，VLAN内的主机可以二层互通，而VLAN间的主机不能二层互通，可以在三层交换机上通过VLANIF来实现VLAN间的三层互通。但如图所示，当三层交换机的三层接口有限时，而三层接口又无法关联VLANIF。此时可将一个三层以太网接口虚拟成多个逻辑子接口。

![fig_dc_cfg_vlan-termination_000201](E:\Photo_Wall\fig_dc_cfg_vlan-termination_000201.png)

而三层以太网子接口不支持VLAN报文，当它收到VLAN报文会丢弃，所以需要在子接口上将VLAN Tag剥离掉，也就是需要VLAN终结。

###	GVRP 

####	定义

GARP 协议主要是用于建立一种属性传递扩散机制。GVRP 是 GARP 的一种应用，用于注册和注销VLAN属性。

####	目的

如果需要为网络中的所有设备都配置VLAN，就需要管理员在每台设备上分别进行手工添加。如果遇到组网复杂，管理员无法短时间了解网络拓扑，或者是网络VLAN太多，工作量会非常大且容易出错。在这种情况下，用户可以通过GVRP的VLAN自动注册功能完成VLAN的配置。

####	基本概念

注册模式：

手工配置的VLAN称为静态VLAN，通过GVRP协议创建的VLAN称为动态VLAN。GVRP有三种注册模式：

- Normal模式：允许动态VLAN在端口上进行注册，同时会发送静态VLAN和动态VLAN的声明信息。
- Fixed模式：不允许动态VLAN在端口上注册，只发送静态VLAN的声明信息。
- Forbidden模式：不允许动态VLAN在端口上进行注册，同时删除端口上除了VLAN1外的所有VLAN，只发送VLAN1的声明信息。



####	配置示例



```bash
# 全局使能GVRP
```

###	VCMP

####	定义

VLAN集中管理协议VCMP（VLAN Central Management Protocol）是位于二层的通信协议。它提供了二层网络中传播VLAN配置信息的功能。

####	目的

大型企业网中，交换机很多，会有大量的VLAN信息需要配置和维护，如果仅靠网络管理员手工操作，工作量很大，也不能保障配置的一致性。

为了解决上述问题，可通过VCMP实现VLAN的集中管理，这样，只需在一台交换机上进行创建，删除VLAN等操作，这些变更会自动通知到指定范围的所有交换机，从而使这些交换机无需手工操作即可实现VLAN的创建、删除等动作的同步，即减少了在多台交换机上修改同一个数据的工作量，也保证了修改的一致性。

####	基本概念

VCMP使用域来管理交换机，用角色来确定设备的属性。

| 角色        | 定义及作用                    | 说明                                                         |
| ----------- | ----------------------------- | ------------------------------------------------------------ |
| Server      | VCMP 管理域的管理角色         | Server 上创建、删除和修改VLAN会在全域传播                    |
| Client      | VCMP 管理与的被管角色         | Client 上创建、删除和修改VLAN不会在域内传播，会被Server发送的信息覆盖 |
| Transparent | 透传角色，透明状态            | Transparent仅仅只会转发VCMP报文                              |
| Silent      | VCMP 管理域的边缘，用来隔离域 | Silent 收到 VCMP报文后直接丢弃                               |



####	配置案例

![fig_dc_cfg_vcmp_001702](E:\Photo_Wall\fig_dc_cfg_vcmp_001702.png)

由于 VCMP报文只能在 Trunk 和 Hybrid 类型接口上传送，为免去手工配置链路类型的麻烦，部署VCMP时，一般需要同时部署LNP动态协商链路类型，最大程序简化用户配置。

1. 配置通过LNP自动协商链路类型

   缺省情况下，全局和接口上的LNP处于使能状态。可以使用 `display lnp summary` 

2. 指定各设备的角色

   ```bash
   [AGG] vcmp role server
   [ACC1] vcmp role silent
   [ACC2] vcmp role client
   [ACC3] vcmp role client
   ```

3. Server和Client上配置VCMP相关参数

   ```bash
   # 在AGG上配置VCMP管理域、设备ID和认证密码。
   [AGG] vcmp domain vdl
   [AGG] vcmp device-id server
   [AGG] vcmp authentication sha2-256 password Hello
   # 在ACC2上配置VCMP管理域和认证密码。
   [ACC2] vcmp domain vd1
   [ACC2] vcmp authentication sha2-256 password Hello
   # 在ACC3上配置VCMP管理域和认证密码。
   [ACC3] vcmp domain vd1
   [ACC3] vcmp authentication sha2-256 password Hello
   ```

4. 使能VCMP功能

   ```bash
   # 缺省情况下，接口上的VCMP功能已使能，无需再使能。但为避免VCMP报文影响PC终端，可Client连接PC终端接口上去使能 
   # VCMP 功能
   [ACC2] interface GigabitEthernet 0/0/2
   [ACC2-GigabitEthernet0/0/2] vcmp disable
   [ACC2-GigabitEthernet0/0/2] quit
   [ACC3] interface GigabitEthernet 0/0/2
   [ACC3-GigabitEthernet0/0/2] vcmp disable
   [ACC3-GigabitEthernet0/0/2] quit
   ```

5. 验证配置

   ```bash
   [AGG] display vcmp status
   VCMP information:
   Domain                  : vd1
   Role                    : Server
   Server ID               : server
   Configuration Revision  : 0x239c0000
   Password                : ******
   ```
