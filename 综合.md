

###	特殊的网络地址段：100.64.0.0/10 

​	这个网段定义在 RFC 6598 中，有别于RFC 1918 的私有地址；它是运营商级NAT（CGNAT）专用的共享地址空间，介于私有网络和公网之间。它是运营商管理维护的，在用户的路由器中，会把它当成一个普通的公网地址来对待。

​	核心作用：使用100.64.x.x 的运营商内部地址，数据包要经过两次NAT才能访问公网。这样的结构也被称为“大内网结构”。

###	Shell的编写规范

####	使用哪种Shell

Bash是唯一被允许执行的shell脚本语言。

可执行文件必须以`#!/bin/bash` 开始，并使用 `set` 来设置 shell 的选项，确保直接执行和显示调用都能有效执行。

####	何时使用Shell

Shell应该仅仅被用于小工具或者是简单的脚本。

以下是一些准则：

- 如果脚本长度会超过100行，应该使用Python来编写。
- 如果脚本需要频繁使用到数组，说明Shell无法优雅处理，应用使用Python来编写。
- 如果在乎性能，应该选择其他工具，而不是Shell。

####	文件扩展名

可执行文件应该没有扩展名，或者使用 .sh 扩展名。

####	SUID/SGID

shell脚本应该禁止使用 SUID 和 SGID

####	标准错误输出

所有错误信息应该输入到标准错误输出中（STDERR）。

下面这个函数是用于打印出错误信息以及其他状态信息的功能，值得推荐：

```bash
err() {
	echo "[$(date +'%Y-%m-%dT%H:%M:%S%z')]: $@" >&2
}

if ! do_something; then
	err "Unable to do_something"
	exit "${E_DID_NOTHING}"
fi
```

####	函数注释

绝大部分函数都需要有注释，任何库函数，无论其长度大小和复杂性都必须要有注释。

函数注释应该包含以下内容：

- 函数的描述信息
- 使用和修改的全局变量
- 参数信息
- 返回值

例子：

```bash
#!/bin/bash
#
# Perform hot backups of Oracle databases.

export PATH='/usr/xpg4/bin:/usr/bin:/opt/csw/bin:/opt/goog/bin'

#######################################
# Cleanup files from the backup dir
# Globals:
#   BACKUP_DIR
#   ORACLE_SID
# Arguments:
#   None
# Returns:
#   None
#######################################
cleanup() {
  ...
}
```

####	格式

缩进：按照2个空格来缩进，不使用tab来缩进。

在两个语句块中间使用空白行来提高可读性。

####	行长度和长字符串

一行的长度不能过长，尽量最多是80个字符。

如果必须写一个长于80个字符的字符串，应该尽量使用here document 或者嵌入一个新行。

```bash
# DO use 'here document's
cat <<END;
I am an exceptionally long
string.
END

# Embedded newlines are ok too
long_string="I am an exceptionally
	long string."
```

####	变量扩展

变量用括号括起来，推荐使用 `"${var}"` 而不是 `"$var"`

单个字符的shell特殊变量则不需要使用括号。

```bash
while read f; do
	echo "file=${f}"
done < <(ls -l /tmp)

# <(command)，进程替换，它会运行括号里的命令，并把该命令的输出结果变成一个临时文件或文件描述符，然后返回这个文件描述符的路径（类似/dev/fd/63）
# 不用管道的原因是，管道会开启子shell, while 会运行在子shell里，循环里修改的变量，定义的函数退出循环后就全部丢失了
```

####	管道导向while循环（Pipes to While）

优先使用进程替换或者for循环，而不是管道给while循环，在while中修改的变量是不能传递给父shell的，因为循环命令是在一个子shell中运行的。



####	主函数

对于足够长的脚本来说，至少需要一个名为main的函数来调用其他的函数。



###	SDH和ATM

####	SDH 同步数字体系

核心思想：统一时钟，刚性管道

它的设计初衷是取代早期的准同步协议，解决不同厂家设备互联困难，上下电路麻烦的问题。

- **严格同步**：全网设备使用同一个高精密度主时钟，所有人步调一致，为数据传输提供极高的可靠性。
- **刚性管道**：带宽被切分成固定大小的时间片，每个用户独占属于自己的时间片，即使不传数据，别人也无法占用。
- **优点**：可靠性极高，管理开销丰富，非常适合承载语音这类恒定速率的核心业务。
- **结局**：它曾是传输网的绝对核心，但因刚性管道的特性，承载突发的IP数据包时效率很低。如今已逐步被OTN（光传送网）取代，它的帧结构思想仍影响着现在的网络。

####	ATM 异步传输模式

核心思想：统一信元，统计复用。

它的目标是融合语音，数据，视频等多种业务到一张网上，用一个统一的格式传输。

- **技术基础**：把一切数据都切成固定 53 字节的“信元”，5字节头加48字节数据。
- **面向连接**：通信前需建立虚电路（VC），保证服务质量（QoS），能为每个“通话”预留带宽。
- **统计复用**：带宽不固定划分，用户有数据就发，没数据就空出信元给其他人。相比SDH的排队，它像繁忙的十字路口，谁有急事谁先过，利用率高。
- **优点**：有精细的QoS控制，能同时很好承载语音、视频和数据。
- **结局**：协议太复杂、53字节信元效率低。当简单高效的IP技术结合MPLS也能提供QoS后，ATM就被淘汰了。不过，它的一些控制思想被MPLS继承，且至今仍是无源光网络（PON）宽带接入的二层协议基础。





####	其他相关名词

- **OTN (光传送网)**：SDH的现代光网络升级版，继承了其运维管理优势，并加入FEC纠错能力，专为大颗粒业务而生，是当前骨干网的核心。
- **WDM/ DWDM (波分复用/密集波分复用)**：把光纤容量挖掘到极致的“光层”技术。SDH/OTN是业务组织者，WDM则是高速公路，在一根光纤里同时跑几十到上百个波长。
- **PTN (分组传送网)**：SDH向IP化的妥协产物，保留了SDH的运维习惯，但内核是MPLS-TP，专为承载IP基站回传设计。
- **IP/MPLS (基于标记交换的IP网)**：IP像写清地址的明信片，MPLS则像给明信片贴标签，让路由器只根据简单标签高速转发。它是融合语言、视频、数据于一身的最终胜出者，也是SRv6等新型广域网技术的源头。

SDH代表了高可靠的硬管道，ATM是勇敢但过早的融合尝试。现在的网络架构通常是：
**数据中心内部**用纯IP，**城域与骨干网**跑IP/MPLS，**底层光缆**用WDM/OTN，而**接入网**则藏着ATM的影子。

















####	PON 无源光网络

无源光网络是一种点对多点的光纤接入技术，名字里的无源是指从运营商的局端设备到家里的这段光纤线路上，没有任何需要用电的电子设备，全靠无源的光分路器（分光器）来分配光信号。

PON的组成：

- OLT（光线路终端）：装在运营商机房，是PON的大脑，负责调度数据，分配带宽。
- ODN（光分配网络）：就是途中的光纤和那个无源分光器，全程无需供电，可靠性很高。
- ONU/ONT（光网络单元/终端）：用户家里的光猫，负责接收光信号。

主流类型：

PON主要是带宽和封装方式不同：

- GPON（千兆无源光网络）：目前国内家庭宽带主流。下行 2.5Gbps，上行 1.25Gbps，所有用户共享。
- EPON（以太网无源光网络）：基于以太网标准，上下行对称 1Gbps，常见于早期网络或对上下行对称要求高的场景。
- 10G PON（万兆无源光网络）：包括 10G GPON 和 10G EPON，带宽提升到万兆，是“千兆宽带”的关键。
- 50G PON：下一代标准，为万兆入户和 5G 承载做准备。

GPON 在二层封装时采用了与 ATM 完全相同的 **GEM（GPON封装方法）** 帧格式，可以看成是 ATM“固定长度信元”思想为适应大带宽业务做的现代改进版。这算是 ATM 技术理念在现代宽带接入网里的延续。



###	IPLC、IEPL、MPLS、SD-WAN

![IPLC、IEPL和MPLS VPN的对比分析](https://www.vecloud.com/uploads/20200324/15849811306642.jpg)

**IPLC国际网络专线：**通过私有线路连接不同地区的网络专线，是一种连接两个或两个以上网络节点，通过这些节点进行点对点数据传输的专线电路。它的优点是能提供独占的带宽，确保数据传输的可靠性和安全性，同时具备较低的延迟；而缺点是建设费用、租用费用和维护费用高，部署周期长，也受地理位置的限制。适用于预算多，对数据保密传输要求高的企业。

**IEPL国际以太网专线：**
